# AGENTS.md — getgrowth-content-engine

This file is the authoritative workspace guide for AI agents and human contributors.

## Repo Purpose

Agentic ASO content pipeline. Takes structured app audit data from Supabase and automatically
generates blog posts, LinkedIn posts, X threads, and email snippets via 5 n8n workflows
powered by Gemini 2.0 Flash.

## Repo Map

```
getgrowth-content-engine/
├── schemas/              ← JSON schemas for agent inputs/outputs
│   ├── audit.json        ← Audit agent output schema
│   ├── blog_post.json    ← Teardown Writer output schema
│   └── ...
├── templates/            ← Prompt templates for each agent
│   ├── audit_prompt.md
│   ├── teardown_prompt.md
│   ├── repurposer_prompt.md
│   └── ...
├── README.md             ← System overview
└── ARCHITECTURE.md       ← Full runbook + Supabase table definitions
```

## Architecture — 5 Agents

| Agent | Trigger | Output | n8n workflow |
|---|---|---|---|
| Audit Agent | Wed 8AM cron | Structured audit JSON → Supabase | `1-audit-agent` |
| Teardown Writer | New audit in Supabase | 1,500-word blog post (markdown) | `2-teardown-writer` |
| Repurposer | New blog post | LinkedIn × 2 + X thread + email snippet | `3-repurposer` |
| Benchmark Aggregator | 1st of month | Category benchmark article | `4-benchmark` |
| Publisher | Daily 10AM M-F | Approved content → CMS + social | `5-publisher` |

## Stack

| Layer | Tech |
|---|---|
| Orchestration | n8n (hosted on deltabyetoro.app.n8n.cloud) |
| Database / state | Supabase (PostgreSQL) |
| AI generation | Gemini 2.0 Flash |
| CMS | Ghost / WordPress (configurable) |
| Social scheduling | Buffer (LinkedIn, X) |

## No Local Dev Server

This repo is **configuration and schema only** — no code runs locally. All agents live in
n8n. This repo tracks:
- Prompt templates (edit here, paste into n8n)
- JSON schemas (used to validate agent outputs)
- Architecture documentation

## Naming Conventions

- Prompt templates: `templates/<agent_name>_prompt.md` (snake_case)
- Schemas: `schemas/<output_type>.json` (snake_case)
- n8n workflow exports (if added): `workflows/<name>.json` — **never export with embedded credentials**

## Do Not Touch

- Do not commit `.env` or any file containing n8n/Supabase/Gemini API keys
- Workflow JSON exports must be credential-free (export as "workflow only" in n8n)

## Supabase Tables

Key tables (defined in ARCHITECTURE.md):
- `app_pool` — curated apps to audit
- `audits` — structured audit results
- `content_pieces` — generated content (blog, social, email)
- `agent_logs` — run history and errors
